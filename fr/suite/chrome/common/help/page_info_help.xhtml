Afficher les informations d'une page

La boîte de dialogue &pageInfoCmd.label; est constituée de plusieurs onglets affichant différentes informations sur la page que vous êtes en train de visualiser. Cette section vous fournit un bref aperçu des informations disponibles dans chaque onglet.

Pour voir les informations de la page présentement affichée par le navigateur, ouvrez le menu &viewMenu.label; et sélectionnez &pageInfoCmd.label;. Vous pouvez sélectionner l'onglet correspondant à l'information recherchée.

La plupart des informations affichées dans les onglets &generalTab;, &formsTab;, &linksTab; et &mediaTab; de la boîte de dialogue sont extraites de la page source. Ces informations intéressent habituellement les développeurs Web et autres spécialistes.

Pour de plus amples informations à propos des langages Web (HTML, XHTML ou encore XML), y compris les balises affichées par les informations sur la page, voir les pages Web :

  • les spécifications HTML 4.01.
  • les spécifications XHTML 1.0.
  • le langage XML.
  • les spécifications CSS1.
  • les spécifications CSS2.1.

L'onglet &securityTab; intègre les informations de la plate-forme pour les préférences de confidentialité (P3P) de la page que vous visualisez présentement et l'état de la sécurité de cette page.

Dans cette section :

  • &generalTab;
  • &formsTab;
  • &linksTab;
  • &mediaTab;
  • &securityTab;

&generalTab;

Lorsque vous choisissez d'afficher les informations sur la page depuis le menu &viewMenu.label;, l'onglet &generalTab; affiche les informations de base de la page en cours.

La partie supérieure affiche le titre de la page (si elle en possède un) et les informations suivantes :

&generalURL;
L'URL de la page est son adresse standardisée qui apparaît dans la barre d'adresse située dans le haut de la fenêtre du navigateur.
&generalType;
Le type MIME du document. Par exemple, pour une page Web vous aurez text/html.
&generalMode;
Indique si le navigateur utilise le mode souple (mode de compatibilité) ou rigide (mode de respect strict des standards) pour afficher la page. Le mode compatibilité tient compte des comportements non standards qui peuvent encore être utilisés par d'anciennes pages Web proprammées pour les anciennes versions de navigateurs Web qui ne respectent pas totalement le standard. Le mode de respect strict des standards applique strictement les spécifications de ceux-ci. Votre navigateur choisit le mode de représentation automatiquement, par rapport aux informations contenues dans la page elle-même.
&generalSource;
Indique si le code source de la page est présent dans le cache.
&generalEncoding;
L'encodage de caractères utilisé pour le document HTML.
&generalSize;
La taille du fichier, si elle disponible.
&generalReferrer;
La page précédente qui vous a permis d'accèder à la page actuelle.
&generalModified;
La date à laquelle la page a été modifiée, si elle est disponible.

La partie inférieure affiche les métabalises définies par la page. Ces métabalises fournissent des informations à propos du contenu de la page comme la description générale de la page, les mots-clés définis pour les moteurs de recherche, les informations de Copyright - Droits d'auteur, etc.

&formsTab;

Lorsque vous choisissez d'afficher les informations sur la page depuis le menu &viewMenu.label;, en ouvrant l'onglet &formsTab;, vous verrez les informations du formulaire contenu dans la page Web.

La partie supérieure affiche les informations de base par rapport à la façon dont est spécifié chacun des formulaires dans le code HTML source de la page :

&formName; :
Le nom du formulaire, s'il existe.
&formMethod; :
La méthode HTML employée pour envoyer l'information mémorisée dans le formulaire vers le serveur Web. La méthode GET ajoute les informations que vous avez données à l'adresse du site Web vers lequel vous soumettez le formulaire. La méthode POST envoie les valeurs au site Web en tant que paramètres pouvant être lus par un programme du site Web.
&formAction; :
L'URL du programme utilisé lorsque le formulaire est soumis.

En sélectionnant un formulaire dans la partie supérieure de l'onglet Formulaire, la partie inférieure affiche de façon détaillée comment ces données sont définies dans le code HTML source de la page :

&formLabel;
L'étiquette de la donnée (s'il existe une telle étiquette).
&formFName; :
Le nom de la donnée.
&formType; :
Le type de donnée en saisie, comme TEXT (pour envoyer du texte), RADIO (pour un bouton radio) ou HIDDEN (pour enregistrer de l'information qui sera soumise, mais pas affichée à l'écran).
&formCValue; :
La valeur donnée à la donnée. Par exemple, la valeur courante d'une donnée de type TEXT est le texte qu'elle contient actuellement.

Si des colonnes ne sont pas visibles, cliquez sur l'icône , puis choisissez les colonnes à afficher.

&linksTab;

Lorsque vous choisissez d'afficher les informations sur la page depuis le menu &viewMenu.label;, en ouvrant l'onglet &linksTab;, vous voyez la liste des liens disponibles sur la page. Les informations suivantes sont affichées pour chaque lien :

&linkName; :
Le texte affiché dans le navigateur pour le lien.
&linkAddress; :
L'URL de la page vers laquelle pointe le lien.
&linkType; :
Le type de lien, tel que les Marqueurs nommés/Ancres (pour un lien vers un endroit précis dans un document HTML ou l'envoi de formulaire).
&linkTarget; :
La cible du lien, fenêtre actuelle ou nouvelle fenêtre.
&linkAccessKey; :
La touche d'accès associée à ce lien, si elle existe.

Si des colonnes ne sont pas visibles, cliquez sur l'icône , puis choisissez les colonnes à afficher.

&mediaTab;

Lorsque vous choisissez d'afficher les informations sur la page depuis le menu &viewMenu.label;, en ouvrant l'onglet &mediaTab;, vous voyez la liste des images et autres données multimédia affichées par la page.

La partie supérieure liste les informations de base pour chaque élément, incluant son adresse (URL), son type MIME et le texte alternatif.

En sélectionnant un élément listé dans la partie supérieure, la partie inférieure affiche les informations disponibles telles que la description, la taille ou les dimensions.

De même, vous pouvez voir l'élément sélectionné dans le bas de la boîte de dialogue. Pour voir les grandes images, vous devez redimensionner la fenêtre en cliquant sur le coin en bas à droite de la fenêtre et en glissant-déplaçant la souris.

Pour enregistrer un élément Média dans un fichier :

&mediaSaveAs; :
Sélectionnez l'élément à enregistrer, cliquez sur &mediaSaveAs;, et choisissez l'emplacement où vous souhaitez enregistrer.

Si des colonnes ne sont pas visibles, cliquez sur l'icône , puis choisissez les colonnes à afficher.

&securityTab;

Lorsque vous choisissez d'afficher les informations sur la page depuis le menu &viewMenu.label;, en ouvrant l'onglet &securityTab;, vous voyez des informations à propos de l'authentification et du chiffrement de la page actuellement affichée. La partie supérieure indique si l'identité du site Web a été vérifiée et la partie inférieure indique si la page a été chiffrée lorsqu'elle a été reçue par le navigateur.

Vous pouvez aussi ouvrir l'onglet &securityTab; en cliquant sur l'icône cadenas située dans le coin en bas à droite de chaque fenêtre d'un navigateur, ou .

La partie supérieure de l'onglet &securityTab; peut afficher l'un de ces messages :

Identité du site Web non vérifiée :
Le site Web que vous visitez n'a pas présenté de certificat l'authentifiant. Par conséquent, le gestionnaire de certificats ne peut pas vérifier son identité. Cependant, il se peut, quoique ce soit improbable, que le site Web ne soit pas celui qu'il prétend être.
Identité du site Web vérifiée :
Le certificat que le gestionnaire de certificats a utilisé pour vérifier l'identité du site a été émis par une autorité de certification (AC) indiquée comme ayant votre confiance. Vous pouvez raisonnablement croire que le site Web est bien ce qu'il prétend être. En cliquant sur Voir, vous pourrez visualiser le certificat de sécurité de la page Web visitée.

La partie inférieure de l'onglet &securityTab; peut afficher l'un de ces messages :

Connexion non chiffrée :
Il est possible qu'un tiers puisse voir les informations envoyées depuis votre ordinateur vers le site Web, et vice versa, mais il est improbable que quelqu'un le fasse vraiment.
Connexion chiffrée :
En général, la force du chiffrement dépend de la longueur des clés utilisées pour le codage, mesurées en bits. Plus la clé est longue, plus fort est le codage et, de fait, plus il sera difficile à une personne non autorisée de décoder une information chiffrée.

La fenêtre &pageInfoCmd.label; décrit la force du chiffrement selon l'un de ces trois cas :

Chiffrement de haut niveau :
Le chiffrement le plus puissant disponible, utilisant des clés d'au moins 128 bits.
Niveau moyen de chiffrement :
Un chiffrement intermédiaire, utilisant des clés de 56 ou 64 bits.
Chiffrement de bas niveau :
Le plus faible niveau de chiffrement, utilisant des clés de 40 bits.

La plupart des sites Web utilisent le niveau élevé de chiffrement. Si vous visitez un site utilisant un niveau bas de chiffrement, il est possible qu'un tiers puisse voir les informations envoyées depuis votre ordinateur vers le site Web, et vice versa, mais il est improbable que quelqu'un le fasse vraiment.

L'absence ou un bas niveau de chiffrement des informations doit vous inquiéter si vous visualisez ou envoyez des informations à caractère confidentiel, tel que votre numéro de carte de crédit, par exemple.

Vous pouvez rapidement vérifier le niveau de sécurité de la page que vous êtes en train de visualiser en vérifiant l'état de l'icône en forme de cadenas dans le coin en bas à droite de la fenêtre de votre navigateur, ou . Pour plus d'informations, voir la section Vérifier la sécurité d'une page Web.
