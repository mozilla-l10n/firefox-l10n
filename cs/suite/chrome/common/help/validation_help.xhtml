Nastavení ověřování platnosti

Tato kapitola popisuje nastavení předvoleb ověřování platnosti a správu seznamu uložených certifikátů CRL.

Pro podrobný popis různých úkolů týkajících se ověřování platnosti a seznamů uložených certifikátů CRL krok po kroku se podívejte na Jak ověřování platnosti funguje.

Předvolby Soukromí & Zabezpečení - Ověřování

Tato sekce popisuje použití podkategorie Ověřování. Pokud dosud nemáte podkategorii zobrazenu, proveďte následující kroky:

  1. Otevřete nabídku &brandShortName; Úpravy a klepněte na položku Předvolby.
  2. V kategorii Soukromí & Zabezpečení klepněte na podkategorii Ověřování. (Pokud nevidíte žádnou podkategorii, dvojklepnutím na Soukromí & Zabezpečení zobrazíte jejich seznam.)

Pro další informace o ověřování platnosti se podívejte do kapitoly Jak ověřování platnosti funguje.

CRL

CRL (certificate revocation list) je seznam uložených certifikátů, které jsou vytvářeny a podepisovány Certifikační autorita (CA).Je možno stáhnout CRL do vašeho Prohlížeče, který jej může kontrolovat a zajistit platnost certifikátů před povolením jejich použití pro ověřování.

Pro zobrazení seznamu CRL dostupných ve Správě seznamu CRL klepněte na tlačítko Správa seznamu CRL.

Více informací o správě CRL seznamů naleznete v kapitole Správa CRL.

OCSP

Protokol OCSP (protokol pro online ověřování stavu certifikátů) umožňuje Správci certifikátů online kontrolovat platnost certifikátu při každém prohlížení nebo použití tohoto certifikátu. Tento proces zahrnuje konfrontaci certifikátu se seznamem uložených certifikátů CRL udržované příslušným serverem. Aby OCSP fungovalo, musí být počítač online.

Pro upřesnění použití OCSP Správcem certifikátů vyberte jednu z následujících možností nastavení v sekci OCSP v podkategorii Ověřování:

  • Používat protokol pro online ověření certifikátů (OCSP) k ověření platnosti certifikátů: Select this setting if you want Certificate Manager to perform an online status check each time it verifies a certificate. If this setting is off, Certificate Manager will only confirm the certificate's validity period and that it is correctly signed by a CA whose own CA certificate is both listed under the CA Certificates tab (in the main Certificate Manager window) and marked as trusted for issuing that kind of certificate.
  • Ověřovat certifikáty, které mají zadán OCSP server: Select this setting for online validation of certificates that provides a validation service address (Service URL). Certificate Manager makes sure that the certificate is listed as valid at the URL and checks validity period and trust settings.
  • Ověřovat všechny certifikáty pomocí tohoto OSCP serveru: Select this setting if you want to specify a OCSP server for the validation of all certificates. If you select this setting, you should also choose the certificate from the Response Signer pop-up menu that identifies the signer of the OCSP responses. With this setting, the only certificates Certificate Manager recognizes are those that can be verified by an OCSP response signed with the Response Signer certificate (or signed using a certificate that chains to it).

    Pokud vybíráte certifikát Podepisovatele odpovědi z rozbalovacího seznamu, Správce certifikátů vyplní editační pole Adresa služby automaticky. Pokud není editační pole Adresa služby vyplněn automaticky, musíte ji uvést sami. Podrobnosti vám sdělí správce vašeho systému.

  • Pokud se připojení k OCSP serveru nezdaří, považovat certifikát za neplatný: Select this if you want the validation to fail if a connection to the OCSP server can't be established.

Správce CRL

Tato sekce popisuje použití Správce CRL. Pro jeho spuštění postupujte následovně:

  1. Otevřete nabídku &brandShortName; Úpravy a klepněte na položku Předvolby.
  2. V kategorii Soukromí & Zabezpečení klepněte na podkategorii Ověřování. (Pokud nevidíte žádnou podkategorii, dvojklepnutím na Soukromí & Zabezpečení zobrazíte jejich seznam.)
  3. Klepněte na tlačítko Správa seznamu CRL.

Otevře se vám dialogové okno Správa seznamu CRL obsahující seznam položek certificate revocation list (CRL), které jste stáhli vaším Prohlížečem. Aktualizujte CRL klepnutím na URL. Pro více informací nahlédněte do kapitoly Správa CRL.

Zvolte položku CRL. Nyní můžete provádět následující operace:

  • Smazat: Smaže natrvalo dané CRL z vašeho pevného disku. Před smazáním se vždy ubezpečte, že toto CRL už opravdu nebudete potřebovat pro ověřování certifikátů. Jestliže si nejste jisti, zeptejte se vašeho systémového administrátora.
  • Nastavení: Otevře se dialogové okno Předvolby automatické aktualizace CRL, které vám umožní nastavit automatické aktualizace pro vybrané CRL s upřesněním, jak často má být prováděna.
  • Aktualizovat: Aktualizuje okamžitě zvolené CRL (pokud je to možné).

O každé CLR poskytuje Správa seznamu CLR následující informace:

  • Organizace (O): Jméno organizace, která CRL vydala.
  • Jednotka organizace (OU): Jméno jednotky organizace, která vydala CRL (podobně jako kořenová CA pro speciální druh certifikátu.
  • Datum poslední aktualizace: Datum, kdy byla provedena poslední aktualizace vaší kopie CRL.
  • Následující aktualizace: Datum, kdy bude vydána aktualizovaná verze CRL.
  • Automatická aktualizace: Příznak, zda je povolena automatická aktualizace CRL. Pro nastavení automatické aktualizace, vyberte CRL a klepněte na tlačítko Nastavení.
  • Stav automatické aktualizace:
    • Pokud automatická aktualizace nebyla nikdy povolena nebo je-li nyní povolena, ale další plánovaná aktualizace ještě neproběhla, bude toto pole prázdné.
    • Poté, co se provedla nejméně jedna automatická aktualizace, zobrazuje toto pole stav poslední takovéto aktualizace — tj. zobrazuje Selhání, jestliže poslední aktualizace selhala nebo OK, jestliže poslední aktualizace byla úspěšná.

CRL Import Status

Tato kapitola popisuje použití dialogového okna CRL Import Status, které se objeví při prvním pokusu importu CRL nebo po úspěšné ruční aktualizaci.

Toto dialogové okno vás informuje

  • zda byl pokus importovat nebo aktualizovat CRL úspěšný
  • jaká organizace vydala CRL
  • kdy bude uvedena příští aktualizace tohoto CRL
  • zda je pro toto CRL povolena automatická aktualizace

Není-li automatická aktualizace povolena, můžete ji povolit zde:

  • Ano: Klepněte na tlačítko Ano pro povolení automatické aktualizace. Po klepnutí se objeví dialogové okno Automatic CRL Update Preferences. Další kapitola popisuje nastavení těchto předvoleb.
  • Ne: Klepněte na tlačítko Ne přejete-li si nechat automatickou aktualizaci zakázanou.

Předvolby automatické aktualizace

Tato sekce popisuje použití dialogového okna Automatic CRL Update Preferences. Toto okno zobrazíte takto:

  1. Otevřete nabídku &brandShortName; Úpravy a klepněte na položku Předvolby.
  2. V kategorii Soukromí & Zabezpečení klepněte na podkategorii Ověřování. (Pokud nevidíte žádnou podkategorii, dvojklepnutím na Soukromí & Zabezpečení zobrazíte jejich seznam.)
  3. Klepněte na tlačítko Správa seznamu CRL a vyberte CRL, u které chcete změnit nebo zobrazit nastavení automatické aktualizace.
  4. Klepněte na tlačítko Nastavení.

Zobrazené dialogové okno obsahuje následující nastavení a informace:

  • Zapnout automatickou aktualizaci pro CRL: Tuto možnost vyberte, jestliže chcete povolit automatické aktualizování CRL podle plánu. (Všimněte si, že tuto možnost nemůžete vybrat, pokud CRL nemá specifikováno datum následující aktualizace.)

    Jestliže povolíte automatickou aktualizaci, musíte vybrat jeden z následujících přepínačů:

    • Aktualizace X dní před datem následující aktualizace: Tuto možnost vyberte jestliže chcete, aby základní opakování aktualizace bylo dáno frekvencí, kterou CRL vydavatel uveřejňuje nové verze CRL.
    • Aktualizace každé X dny: Tuto možnost vyberte, jestliže chcete určit interval aktualizace nezávisle na datu následující aktualizace CRL.
  • CRL bude importováno z: Označuje URL, ze které Prohlížeč importoval původní CRL. Tato hodnota nemůže být změněna. Pro změnu umístění, smažte CRL a importujte ho z nového místa.
  • Předcházející aktualizace selhala: Označuje, kolikrát se Prohlížeč pokoušel neúspěšně aktualizovat CRL, včetně posledního nezdaru:
    • Jestliže poslední pokus byl úspěšný, je zobrazen řetězec Žádný a to i v případě, že předcházející pokusy byly neúspěšné.
    • Jestliže poslední pokus selhal, je zobrazeno číslo, odpovídající počtu nezdarů a chybová zpráva pro poslední nezdar.

Klepněte na tlačítko OK pro potvrzení vašich změn.
